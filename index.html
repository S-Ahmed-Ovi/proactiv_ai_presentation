Chapter 03

The Problem &
Market Gap

Building recommendation infrastructure from scratch is painful, expensive, and slow. Every organization faces the same six obstacles.

Massive Build Time
6–12 months to build recommendation infrastructure from scratch. Requires specialized ML teams most organizations don't have.
Cold-Start Problem
New users and items receive poor recommendations without historical data. Most systems have no fallback strategy.
Model Drift
Performance degrades over time as user preferences shift. No automated retraining pipelines exist in off-the-shelf solutions.
Domain Mismatch
One-size-fits-all models fail across verticals. A movie recommender and an e-commerce engine need fundamentally different approaches.
Multi-Tenancy Afterthought
Most systems lack per-tenant data isolation and model versioning. Cross-tenant data leakage is a real regulatory risk.
Discovery Friction
Users cannot find relevant items. Stale recommendations don't reflect evolving taste. No transparency in why items were recommended.

Chapter 07

Recommendation
Pipeline Deep Dive

Seven stages from raw data to live recommendations. Data flows through each transformation automatically — no manual intervention at any step.

1
Data Ingestion
CSV upload or SQL connector → pandas DataFrame. DataCleaner validates columns (userID, itemID, rating/timestamp), drops nulls, deduplicates.
2
Feedback Detection
FeedbackDetector auto-classifies: explicit (ratings 1–5 present) | implicit (binary interactions) | both. Drives algorithm eligibility filter.
3
Interaction Matrix
InteractionMatrixBuilder constructs scipy.sparse CSR matrix. user_index and item_index dicts for O(1) lookup. 80/20 train/test split.
4
Embedding Step ✨
EmbeddingEnricher: Sentence-BERT all-MiniLM-L6-v2 → L2-normalized 384-dim vectors → bulk upsert into OpenSearch HNSW kNN index (per-tenant isolated).
5
Model Training & Benchmark
21 algorithms run with Optuna TPE tuning (SQLite-backed, per-tenant study). BenchmarkEngine evaluates each on held-out test set.
6
Promotion & Serving
Composite score (70% NDCG + 20% speed + 10% scalability) selects best. /models/promote → registry.json updated. ServingPipeline blends CF score + embedding score (alpha=0.7).

Chapter 10

Automated Model
Selection Engine

Zero manual intervention from data upload to live recommendations. The pipeline discovers, evaluates, and promotes the optimal model autonomously.

1
Feedback Type Detection
FeedbackDetector inspects dataset columns. Explicit = rating column present. Implicit = binary events only. Filters algorithm pool to eligible_feedback ∈ {explicit|implicit|both}. Prevents nonsensical model evaluations.
2
Scale Tier Classification
DatasetAnalyzer counts unique users and items. Tier-A (>10K users or >5K items): downsamples to A limits. Tier-B (>3K/2K): applies lighter downsampling. Prevents OOM errors on dense algorithms like EASE and User-KNN.
3
Algorithm Eligibility Filter
Algorithms with hard max_users/max_items limits are checked against actual dataset dimensions post-downsampling. Ineligible algorithms skipped with recorded skip_reason in training result — full audit trail maintained.
4
Optuna Hyperparameter Tuning
For each eligible algorithm, Optuna TPE sampler runs N trials auto-budgeted by complexity. Best hyperparameters stored in SQLite-persisted study (results/optuna_studies.db). TimeoutCallback limits total tuning to OPTUNA_TIMEOUT_S = 600s.
5
BenchmarkEngine Evaluation
BenchmarkEngine.run() evaluates each algorithm on held-out test set. Computes NDCG@K, Precision@K, Recall@K, RMSE, MAE, training time, and scalability_score. Full leaderboard stored in last_training_result.json.
6
Composite Scoring & Auto-Promotion
composite = 0.7 × normalize(primary_metric) + 0.2 × normalize(speed) + 0.1 × scalability_score. Top-ranked model is automatically promoted to tenants/{id}/models/. Next /recommend request uses the new model instantly — zero manual intervention.
Composite Score = 0.70 × Performance + 0.20 × Speed + 0.10 × Scalability
Why this weighting? Accuracy matters most (70%) but a 3× faster model is worth choosing over a marginally more accurate one (20%). Scale limits (10%) ensure chosen model can handle production data volume.

Chapter 12

Security
Architecture

Defense in depth across every layer — with an honest assessment of current gaps and a concrete path to production hardening.

Implemented Controls
🔑
JWT Authentication
HMAC-SHA256 signed tokens via python-jose. Configurable TTL. In-memory revocation list for token invalidation.
🗝️
API Key Authentication
Per-tenant keys with hash-based comparison. TenantManager.resolve_api_key() maps key to tenant_id — raw keys never stored.
🛡️
Scope-Based Authorization (RBAC)
11 scopes from recommend:read to admin:write. @require_auth(scopes=[...]) decorator on every route enforces per-endpoint authorization.
🏢
Tenant Isolation
All model loads use tenant_id prefix: tenants/{id}/models/. Cross-tenant model access is impossible at the API level.
🔒
BCrypt Password Hashing
passlib[bcrypt] with appropriate work factor. No plain-text credentials stored anywhere in the system.
⚡
Redis Rate Limiting
INCR counter with TTL per API key. Metering middleware tracks per-tenant call counts for billing and abuse detection.
✅
Input Validation
Pydantic v2 validates all request bodies. Custom field_validators enforce business rules at schema level before any business logic executes.
Known Gaps — Roadmap
📁
Secrets in .env File
AUTH_SECRET_KEY and REDIS_PASSWORD in repo .env. Migrate to AWS Secrets Manager, HashiCorp Vault, or GitHub Secrets before production.
💾
In-Memory Token Revocation
_TOKEN_REVOCATION_LIST lost on process restart. Move to Redis with TTL for distributed, persistent revocation across replicas.
🌐
No HTTPS Enforcement
TLS termination not configured in Docker Compose. Add NGINX reverse proxy with Let's Encrypt certificate for production deployment.
💉
LLM-Generated SQL Risk
Smart DB CSV Builder constructs SQL from LLM output. Validate with sqlparse or enforce parameterized queries before execution.
🔄
No API Key Rotation
No built-in key rotation mechanism. Add POST /tenant/rotate-key endpoint with grace period for zero-downtime key rotation.
📱
No MFA
Login uses only username + password. Add TOTP-based MFA for admin:write scope users in regulated industry deployments.
Chapter 13

Product Roadmap

Three phases from production hardening to next-generation AI — each with concrete engineering milestones and clear business outcomes.

🌱
Short Term · 0–3 Months
Production Hardening
  • Prometheus + Grafana observability stack (prometheus-fastapi-instrumentator)
  • Structured JSON logging (python-json-logger → Loki)
  • Parallelize SLIM training (joblib n_jobs=-1) for 5–10× speedup
  • Pre-compute recommendation lists post-training → cache in Redis
  • Add MAP, MRR, Hit Rate, Coverage, Diversity metrics to BenchmarkEngine
  • HTTPS via NGINX reverse proxy + Let's Encrypt certificate
  • API key rotation endpoint + grace period for smooth transitions
  • Migrate secrets to environment-based secrets manager (Vault/AWS)
  • Kubernetes manifests + Helm chart for cloud deployment
  • GitHub Actions CI/CD pipeline: test → build → push → deploy
🚀
Medium Term · 3–9 Months
Platform Scale
  • MLflow / Weights & Biases integration for experiment tracking
  • S3 / GCS backend for tenant model artifacts (replace filesystem)
  • PostgreSQL for tenant metadata, job history, and audit logs
  • A/B testing framework: traffic split between two active model versions
  • Real-time event streaming: Kafka/Kinesis → live interaction ingestion
  • LightGBM re-ranking layer (post-process CF candidates with feature model)
  • Model drift detection: daily NDCG@K evaluation + auto-retrain trigger
  • FAISS in-memory ANN as fallback when OpenSearch is unavailable
  • Multi-armed bandit for exploration-exploitation in recommendations
  • Tenant-level SLA guarantees and quota management dashboard
⭐
Long Term · 9–24 Months
Next-Generation Intelligence
  • Deep learning models: DIN, SASRec, Transformers4Rec for sequential recommendations
  • Graph Neural Network recommendations (PinSage, LightGCN architecture)
  • Cross-tenant federated learning with differential privacy guarantees
  • Real-time API with sub-10ms P99 via pre-computed indexes and edge caching
  • LLM-powered explanation generation: conversational "why was this recommended?"
  • Conversational recommendation via chat interface (Claude / GPT integration)
  • Automated feature discovery from raw event streams (AutoFE)
  • Marketplace: tenants publish and subscribe to domain-specific pretrained models
  • Edge deployment: compressed model artifacts for on-device inference
  • SOC2 Type II, GDPR, and HIPAA compliance certifications
